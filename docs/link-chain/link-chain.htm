Link Chain Bar

The link-chain web component is a horizontally stretching panel that carries links to the previous and following sibling documents, the current folder and the parent folder. It can be used to provide a simple navigation aid for documents on the same level of a tree structure, for instance, pages of a chapter, demos or studies related to a certain topic. The filenames should have a numerical part with increments of one, and identical prefixes and suffixes. Example: "study-01.htm", "study-02.htm", and so on. If the author follows this naming pattern, the link-chain component can extract all the necessary information from the document.location.path.

Demo

See study-01.htm, a chain of three documents that demonstrates use and styling.

Usage

Include the component script in the document head:

<script type="module" src="\path\to\link-chain.js"></script>

Then use the component in the document body:

<link-chain></link-chain>

To indicate that a document is the last one in the sequence, add the no-following attribute:

<link-chain no-following></link-chain>

The first document is automatically recognized if the document number equals one. Otherwise, you can add the no-previous attribute instead:

<link-chain no-previous></link-chain>

Styling: The component publishes the left, center, right and anchor parts so that they can be styled from the host document, as demonstrated in study-01.htm and study-02.htm:

link-chain { font-family : "arial" ; border : 2px solid blue }
link-chain::part( anchor ) { color : red }

Implementation Details

The component constructor checks for the existence of a shadow root. If it exists, the user has supplied a TEMPLATE child element with a shadowrootmode attribute and HTML content. Otherwise, the HTML code is loaded from .../webcat/link-chain/link-chain.htm. The code contains a STYLE element that defines the initial appearance. Some aspects can be tuned with CSS from outside.
