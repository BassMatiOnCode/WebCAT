SPA Host Component

Usage

Using a Preconfigured Demo Template

This template hosts content documents in an IFRAME element.

  1. Copy the spa-host-demo-template.htm to the root folder of your web site, and rename it to index.htm. Note that this filename must be referenced from redirection scripts in content documents.
  2. Add the following files to the site:
    1. logo.svg
    2. title.svg
    3. colors.css
    4. elements.css
    5. toc.htm
  3. Create content documents.

Using Barebone Template

This template also hosts content documents in an IFRAME element.

  1. Copy the spa-host-template.htm to the root folder of your web site, and rename it to index.htm. Note that this filename must be referenced from redirection scripts in content documents.
  2. Edit the document meta information in the document HEAD.
  3. Add components for the static user interface to index.htm, such as page header,, page toolbar, page pathbar, navigation panel.
  4. Add the following files:
    1. logo.svg
    2. title.svg
    3. colors.css
    4. elements.css
    5. toc.htm
  5. Create content documents and add them to the site.

Create from Scratch

This demonstrates how to build an SPA host document from scratch.

  1. Create an empty HTML file, and name it index.htm.
  2. Add the usual META elements in the HEAD section.
  3. Add a LINK element, rel="stylesheet", href="path/to/spa-iframe-host.css".
  4. Add a SCRIPT element, type="module", src="path/to/spa-iframe-host.js".
  5. Add components to create the static interface for page header, toolbar, breadcrumbs list, site navigation panel and others, as desired. The WebCAT project provides a bunch of ready-to-use components, but you can create your own, of course.
  6. Create content documents and add them to the site.

If you want to use a different content document container, use spa-div-host.css and spa-div-host.js for the LINK and SCRIPT elements instead.
