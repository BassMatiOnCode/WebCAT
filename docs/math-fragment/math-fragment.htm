Math Fragment

Additional features for math content fragments

Considerations

Firstly, some definitions:

math fragment loader anchor

Located in a fragment host document, processed by the fragment-loader, and is replaced with a portion of the math fragment document.

math fragment document

A is a self-contained, complete HTML document with HEAD and BODY elements, not just some bare HTML code fragment. Contains an element decorated with the class-name math-fragment-content, which will replace the math fragment loader anchor.

math fragment popup anchor

References a math fragment document and opens it in a separate popup window.

host document link anchor

Resides in the math fragment document and points to the math-fragment-content element in the host document.

The math fragment template serves a dual purpose: a) as stand-alone HTML document to be shown side-by-side to a math page, and b) as HTML fragment content provider. In this case, the FIGURE element with the math-fragment-content class name will be extracted and embedded in a host document.

The math fragment filename must follow a pattern: a lowercase letter (fragment type), followed by a small horizontal bar, a numerical digit sequence (fragment number) and the file type extension (.htm). The filename is used to complete fragment type and number elements the fragment content.

When shown as stand-alone HTML document, the module init code runs over the content and fulfills the following tasks:

  1. Completes the target element ID in the host documemt URL
  2. Fill mathFragmentId and mathFragmentNumber fields
  3. Copy content into elements with a data-select attribute (e. g. the fragment title)

In a stand-alone HTML document, the hostDocument points to the host document where the math fragment is embedded. This information cannot be generated at runtime, and therefore must be hard-coded into the document. On the contrary, the link target id "t-003" can derived from the math fragment document URL ("t-003.htm") and therefore will be generated at runtime. This makes it possible to rename the math fragment document without having to modify the content.

When the math fragment content is injected into a host document, the ID of the FIGURE element must be generated from href (minus .htm suffix) and the optional data-id-prefix attribute in the math fragment loader anchor:

Additionally, the hostDocument anchor must be converted from a host document link to a math fragment link anchor. The ID attribute is removed, and its href must be changed to point to the math fragment document (t-003.htm). A click on this anchor will open the stand-alone math fragment document in a new floating popup window.

In every case, all math fragment loader anchors must listen to the fragment-loader-selecting-content event in order to a) create a math fragment link anchor in the fragment that links to the stand-alone math fragment document, and b) set the id of the math fragment element (usually a FIGURE element).

The fragment container (usually a document.body) must listen to the fragment-loader-root-container-finished event in order to process math fragment link anchors

setup a math fragment link anchor in the math fragment that links to that (stand-alone) math fragment document.

Introduction

This component supports re-usable math equation fragements and provides styles for math equations and shows a title string if the fragment is shown in its own window.

Using the Component

The component consists of two files:

  • math-equation-fragment.css
  • math-equation-fragment.js

These files are used in the HTML fragment that contains the equation.

In a host document, use an HTML OBJECT element to import the fragment:
