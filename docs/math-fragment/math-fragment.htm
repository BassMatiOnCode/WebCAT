Math-Fragment Component

Custom HTML Elements for Math Content Fragments

Definitions, axioms, theorems, corollaries and lemmas frequently appear in mathematical textbooks, and the are often referenced from multiple locations in that text. An example:

The math fragments are stored as bare text in separate files, without head and body.

Overview

The <mfrag-anchor> Element

The <mfrag-anchor> element is another custom HTML element that provides a link to a math fragment. If a related <mfrag-container> is on the same page, the browser window is scrolled to that fragment container. If not, a new browser tab is opened to show the math fragment. A control-click does the same. A shift-click opens the math fragment in a new browser window.

To integrate the math fragment, add a <mfrag-container> element to the host document. Attributes provide the necessary information to build the URL of the math fragment:

The title string has been placed in an link anchor element. A click opens the math content fragment in a separate browser tab. A shift-click opens a new browser window.

Using the Component

1. Add the component script to the main document:

<script type="module" src="path/to/mfrag-container.js"> </script>

If you want to provide your own mfrag-host.htm document, add a search parameter to the URL:

<script type="module" src="path/to/mfrag-container.js?fragment-host=path/to/custom-fragment-host.htm"> </script>

2. Make sure that the associated files are installed in the same directory:

  • mfrag-container.css
  • mfrag-host.htm
  • mfrag-host.css
  • mfrag-host.js

They are referenced implicitly from mfrag-container.js and the default mfrag-host.htm.

3. Make sure that the numbered-equation component (.js and .css files) is installed in the ...webcat/numbered-equation/ folder.

4. Write the HTML fragment that represents the content. Example:

<h1>Pythagorean Theorem</h1>
<div class="content">
<math>
<msup><mi>a</mi><mn>2</mn></msup>
<mo>+</mo><msup><mi>b</mi><mn>2</mn></msup>
<mo>=</mo><msup><mi>c</mi><mn>2</mn></msup>
</math>
</div>

The fragment must have an H1 heading for the title string, and a DIV element with a classname "content". You don't have to provide a complete HTML document, just the bare content is enough, becaus the fragment will be injected into a host document.

5. Save the fragment and supply a filename in the form

fragement-id.htm
fragment-id=descriptive-string.htm

The fragment-id is mandatory and should be unique. It can have its own structure that suffices your needs. A good example is

fragment-type chapter-number.sequence-number

Example:

T1.1=pythagorean-theorem.htm

NOTE that the fragment-id must match the id attribute of the <math-fragement> element. If a descriptive title ("pythagorean-theorem") is included in the filename, that string must appear in the name attribute:

<mfrag-container id="T1.1" name="pythagorean-theorem"></mfrag-container>

Standalone Math Fragment Host

If you click on the fragment ID in the <mfrag-container> in a host document, the content fragment is opened in a separate window. The document provides a full HTML document with HEAD and an empty BODY. The script code loads the content into the document body. A standard implementation can be found in the /inc/webcat/math-fragment folder:

  • math-fragment-host.htm
  • math-fragment-host.css
  • math-fragment-host.js

But you can provide your own implementation if you must.

The math-fragment component in the host document calls the fragment host document with some parameters in the URL search parameters:

fragmentUrl=path/to/the/math-content-fragment.htm
Tells the popup window where to find the content to load.
backlink=path/to/the/host-document.htm
hash=math-fragment-id in the host document

Math Fragment Anchor

A math-fragment-anchor element that points to a mfrag-container element. It works similar to a regular HTMLAnchorElement, but adds a click handler that opens the associated math fragment in a popup window (shift-click) or in a new browser tab (ctrl-click).

This is an example: D 12.3

These pieces result in the URL

studies/03/D 12.3=binomial theorem.htm

The ID attribute is presented as link that opens the math fragment in a separate window. This is useful to keep definitions and theorems open side by side while reading through the main text.

The benefit of storing such content fragments in separate files is that they can be used in multiple chapters, wherever the context recommends it. Another benefit is that the math fragment can be opened in a dedicated, separate window and kept open for reference while reading through the main text. The reader can eliminates the necessity for scrolling back and forth in the text to gain understanding.

The mfrag-container custom element is a container for HTML fragments with math content. The element provides the border, styles and code.

The math fragment URL is compiled from the path, id and name attributes, in this order. The path attribute is optional and can be omitted if the fragment document is in the same directory as the host document. If necessary, a forward slash "/" is appended to the path.

The id attribute uniqely identifies the mfrag-container. The structure of that string is in the responsibility of the author, but the sequence is commonly found in the literature. A letter stands for the fragement type (D=definition, A=axiom, T=theorem, C=corollary, L=lemma, ...), followed by a chapter number and the sequence number.

The id serves a double purpose: Firstly, it serves as part of the math fragment filename, and secondly, it serves as link target (URL hash).

The name attribute is optional, and is a part of the fragment URL if present, separated by an equal sign "=" from the id. The string ".htm" is appended if the filename doesn't end with ".htm" or ".html".

The example above generates the URL "studies/03/D 12.3=binomial theorem.htm". The mfrag-container custom element loads that document and injects it into its shadow tree.

Introduction

This component supports re-usable math equation fragements and provides styles for math equations and shows a title string if the fragment is shown in its own window.
