Study 5 Sender

Sending messages to other windows

Notes:

1. The first message can be sent after a delay of at least 3 animation frame requests.

2. If the receiver window is opened in a new tab, the first message is not displayed before some user interaction takes place, for example, switching back and forth between sender and receiver tab, or tearing off the receiver tab from its parent window.

If the receiver is opened as a popup window, the first message is shown immediately without the need for any user interaction.

This might not happen in other browsers, so the sender window should evaluate the URL search parameters for a flag like "receiver=popup".

3. A refresh of the sender window reloads the receiver document, all messages are lost.

Therefore, the receiver document must maintain a database to keep track of the content.
