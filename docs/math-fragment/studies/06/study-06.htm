Study 6

Comminication via Broadcast Channel

This study examines messaging to a receiver window via BroadcastChannel. The sender (this window) opens the receiver window on the first send request and sends the message after a small delay (to give the receiver window time for starting).

The following send requests ping the receiver window with a "hello" message. If the receiver answers with a "What's up" message, the sender knows that the receiver is up and running, and sends the message. If that message times out, then the receiver window is not (yet) open, therefore the receiver is (re)opened.

If the receiver window is closed, the message are lost.
