Study 7 Sender

Communication With Promises

Send

This study examines messaging to a receiver window via BroadcastChannel. The sender pings the receiver with a "Hello?" message. If the receiver answers with "What's up?", the sender knows that the receiver is up and listening.

If the response does is not received within a second, the receiver is considered to be down and its window is (re)opened. The receiver window sets up its BroadcastChannel and sends "Good morning!" to indicate readyness. When the sender receives this message, the actual message is sent.

If the receiver window is closed, the message are lost.
