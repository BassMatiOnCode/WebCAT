WebCAT

A web content authoring toolbox

WebCAT formalizes the bundling of JavaScript and CSS files, symbolic links and version naming rules into WebCAT components.

WebCAT also introduces the concept of modular HTML documents, document fragments, which enables content sharing bewteen documents. The site navigation tree is a prominent example.

Introduction

Typical WebCAT components are visible, like the page header, footer, path bar, toolbar, tool buttons, navigation panel. Other components are invisible and provide functionality, such as the collapsible structures component, CSS counters, sitemap navigator, page scrolling, page templates and other components. Then there are programming support components, such as the XML namespace or the utility components.

Writing Your Own WebCAT Component

If there is nothing in the box you are looking for, you can easily write your own WebCAT component. The section Creating WebCAT Components has a lot background information and step-by-step instructions.

And of course, existing WebCAT components can serve as a starting point for your own ones.

Overview

A side note: Dividing components into categories is a difficult task. Visual components also have functional aspects, and some functional components have visual aspects.

Visual Components

A visual component creates visible elements on the screen. Examples are page-header, page-footer, page-toolbar and tool-buttons

Functional Components

Functional components have no visible user interface elements, they change the behaviour of HTML elements, add functionality or re-arrange elements. Examples are the page-persistence component, the docking-panel, the page-main content container,
